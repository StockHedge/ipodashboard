# IPO 공모주 대시보드 — 실무진 설명서

> 민트투자자문 내부 공유용. **API 키 포함 패키지** — 외부 유출 금지.

---

## ⚠️ 보안 (가장 먼저)

이 패키지의 **`.env` 파일에는 실제 API 키가 들어 있습니다**:
- `DART_API_KEY` (전자공시)
- `KIS_APP_KEY` / `KIS_APP_SECRET` (한국투자증권 — **모의투자** 모드)
- `KRX_API_KEY` (한국거래소 — 현재 미활성)

**금지 사항**:
- 이 ZIP / `.env` 를 외부(메신저·이메일·클라우드 공개 링크)로 전달 금지
- GitHub 등 공개 저장소 커밋 금지 (`.gitignore` 에 `.env` 등록되어 있으나 ZIP 자체 관리 주의)
- 키 유출 의심 시 즉시 각 콘솔(opendart / KIS / KRX)에서 **폐기·재발급**

키는 **서버(serve.py) 에서만** 사용되며 브라우저로 노출되지 않습니다 (모든 시세 호출은 로컬 프록시 경유).

---

## 1. 설치 (최초 1회)

```powershell
# Python 3.10+ 필요
cd ipo-dashboard
pip install -r requirements.txt
```

## 2. 실행

```powershell
python scripts/serve.py 8000
```
브라우저에서 **http://localhost:8000** 접속.

> 단순 보기만 원하면 `python -m http.server 8000` 도 가능하나, **새로고침·실시간 시세는 serve.py 필수**.

## 3. 데이터 준비 (최초 + 갱신 시)

```powershell
# XLSX → JSON 추출 (Downloads 의 원본 시트 기준)
python scripts/extract_ipo_data.py --xlsx "$env:USERPROFILE\Downloads\새 계정 프로그램 시트 (1).xlsx" --years 3 --out data/ipo-recent.json
```
또는 대시보드 헤더의 **"새로고침"** 버튼 (serve.py 가 자동 실행).

## 4. 화면 구성 (사이드바 5-view)

| 메뉴 | 기능 |
|------|------|
| **대시보드** | KOSPI/KOSDAQ 실시간 + 투자자 순매수, KPI 6종, 차트 4종, 락업 캘린더, 백테스팅, 테이블 |
| **고급 분석** | 히트맵(업종×분기) / 상관 매트릭스 / Violin 분포 / Parallel / 밴드vs확정가 / 락업해제후 |
| **종목 지정** | 회사 참여 종목 체크 + 매수/매도가/메모 → "참여 종목만" 필터 전역 적용 |
| **포트폴리오** | 보유 종목코드 입력 → KIS 실시간 평가손익 |
| **이때팔걸!** | 참여 종목 매도단가 vs 실시간/최고가 비교 → 매도 타이밍 회한 등급 |

**사이드바 도구**: 캘린더 / 즐겨찾기 / 수익률계산기 / 주관사 랭킹 / URL공유 / 데이터

**키보드**: `Ctrl+K` 명령 팔레트 · `1~5` 기간 · `/` 검색 · `t` 테마 · `?` 도움말

## 5. 데이터 소스 현황

| 소스 | 상태 | 비고 |
|------|------|------|
| XLSX 마스터 | ✅ | IPO 228건. 본문 분석의 기준 |
| 네이버 금융 | ✅ (키 불필요) | 지수 + 투자자별 순매수 실시간 |
| KIS (한국투자) | ✅ (모의투자) | 종목 현재가/일별. 실거래 전환은 `.env` `KIS_USE_PAPER=false` |
| KRX OpenAPI | ⚠️ 키 미활성 | openapi.krx.co.kr 콘솔에서 "KRX 시리즈 일별시세" 활용신청 필요 |
| DART | ✅ | 증권신고서 보강 |

> **KRX 활성화**: openapi.krx.co.kr 로그인 → 마이페이지 → 해당 API [활용신청]. 완료 후 코드 수정 없이 작동.

## 6. 자주 묻는 질문

- **마켓 hero 가 "시세 미수신"**: serve.py 가 실행 중인지 확인. `http://localhost:8000/api/ping` 으로 진단.
- **종목명 옆 ↗ 안 보임 / 네이버 안 열림**: ticker 누락 종목 → 자동으로 네이버 "검색" 으로 fallback. 또는 종목 지정에서 종목코드 수동 입력.
- **포트폴리오 종목명 안 나옴**: serve.py 가 fdr 종목 마스터로 자동 보완 (첫 호출 시 수 초 소요).
- **스팩 등 이때팔걸 "종목코드 미등록"**: 입력 필드에 6자리 코드 직접 입력 시 KIS 자동 조회.

## 7. 파일 구조

```
ipo-dashboard/
├── index.html              # 단일 파일 대시보드 (전체 UI + 로직)
├── .env                    # ⚠️ API 키 (외부 유출 금지)
├── .env.example            # 키 placeholder 템플릿
├── requirements.txt        # Python 의존성
├── README.md / MIGRATION.md / 실무진_설명서.md
├── data/                   # 추출 데이터 (gitignore)
└── scripts/
    ├── serve.py            # 로컬 API 서버 (실행 진입점)
    ├── extract_ipo_data.py # XLSX → JSON
    ├── fetch_latest.py     # DART 보강
    ├── naver_client.py     # 네이버 금융 (지수/투자자)
    ├── kis_client.py       # 한국투자 (종목 시세)
    ├── krx_client.py       # 한국거래소 (지수, 키 활성화 대기)
    └── dart_client.py      # 전자공시
```
